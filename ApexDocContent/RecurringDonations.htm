Recurring Donations 1.0 Related Classes

RD1 in NPSP supports two distinct types of Recurring Donations.

  • Open-Ended Recurring Donations - these specify a set of new Opportunities on a recurring schedule
  • Non Open-Ended Recurring Donations - these specify a fixed number of new Opportunities

The classes involved in supporting Recurring Donations are:

  • RD_AddDonationsBTN_CTRL - The page controller for the Refresh Opportunities button.
  • RD_BulkTests_TEST - Test class that contains a test for creating multiple large RD's at once.
  • RD_ProcessControl - Semaphore class that maintains two flags used by RD's.
  • RD_RecurringDonations_BATCH - Batch class that is scheduled to run regularly to process the creation of future Opportunities.
  • RD_RecurringDonations_Opp_TDTM - Trigger Handler on Opportunities to help manage Recurring Donations.
  • RD_RecurringDonations_TDTM - Trigger Handler on Recurring Donations.
  • RD_RecurringDonations_TEST - Test methods that cover all Recurring Donations functionality.
  • RD_RecurringDonations - The main logic for supporting Recurring Donations.

Enhanced Recurring Donations (aka RD2)

The classes involved in supporting Enhanced Recurring Donations are:

  • RD2_Donor - Recurring Donations Donor record and gateway classes
  • RD2_Donor_TEST - Unit Test for the RD2_Donor class
  • RD2_NamingService - Naming Service for automated naming of Recurring Donations records (also supports RD1)
  • RD2_NamingService_TEST - Unit Tests for the RD2_NamingService class

Objects unique to RD2

  • RecurringDonationSchedule__c (This is a hidden system object)
  • RecurringDonationChanges__c
